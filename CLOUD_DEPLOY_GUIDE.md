# 云函数部署指南

## 概述
本项目包含多个云函数，需要在微信开发者工具中上传部署后才能正常使用。本文档说明如何正确部署这些云函数。

## 云函数清单

### 1. getBirdDataBySpot
- **功能**: 获取特定观鸟点的鸟类数据
- **用途**: 根据用户选择的观鸟点获取该区域的鸟类观察记录
- **调用方法**: `wx.cloud.callFunction({ name: 'getBirdDataBySpot', data: { spotName: '深圳湾公园' }})`

### 2. getFeaturedBirdingSpots
- **功能**: 获取深圳代表性观鸟点的鸟类数据
- **用途**: 获取深圳5大代表性观鸟点的综合鸟类观察记录
- **调用方法**: `wx.cloud.callFunction({ name: 'getFeaturedBirdingSpots' })`

### 3. getBirdDataByLocation
- **功能**: 根据地理位置获取鸟类数据
- **用途**: 根据经纬度坐标获取附近区域的鸟类观察记录
- **调用方法**: `wx.cloud.callFunction({ name: 'getBirdDataByLocation', data: { latitude: 22.49, longitude: 113.94, radius: 5 }})`

### 4. getRealBirdData
- **功能**: 获取真实的鸟类观察数据
- **用途**: 获取预设的真实鸟类观察数据
- **调用方法**: `wx.cloud.callFunction({ name: 'getRealBirdData' })`

### 5. addRealBirdData
- **功能**: 添加真实的鸟类观察数据
- **用途**: 用户提交鸟类观察记录
- **调用方法**: `wx.cloud.callFunction({ name: 'addRealBirdData', data: { observation: {...} }})`

### 6. initBirdDatabase
- **功能**: 初始化鸟类观察数据库
- **用途**: 创建数据库集合并添加示例数据
- **调用方法**: `wx.cloud.callFunction({ name: 'initBirdDatabase' })`

### 7. getBirdData
- **功能**: 获取一般鸟类数据
- **用途**: 获取一般鸟类数据（向后兼容）
- **调用方法**: `wx.cloud.callFunction({ name: 'getBirdData' })`

### 8. addBirdData
- **功能**: 添加一般鸟类数据
- **用途**: 添加一般鸟类数据（向后兼容）
- **调用方法**: `wx.cloud.callFunction({ name: 'addBirdData', data: { birds: [...] }})`

## 部署步骤

### 方法一：逐个部署
1. 打开微信开发者工具
2. 在左侧项目目录中找到 `cloudfunctions` 文件夹
3. 展开文件夹，你会看到所有的云函数子文件夹
4. 对于每个云函数文件夹，右键点击并选择 "上传并部署"
5. 选择 "云端安装依赖" 选项

### 方法二：统一部署
1. 在微信开发者工具顶部菜单栏选择 "云开发" → "云函数"
2. 在云函数管理界面中，点击 "上传所有函数"
3. 确认上传所有函数及依赖

## 部署前准备

### 1. 开通云开发
- 在微信公众平台确认已开通云开发功能
- 在项目配置中填写正确的环境ID

### 2. 检查依赖
- 确认每个云函数目录下的 `package.json` 文件正确配置了依赖
- 主要依赖为 `wx-server-sdk`

### 3. 环境配置
- 在项目根目录的 `project.config.json` 中配置正确的云开发环境ID
- 确保 `appid` 配置正确

## 数据库权限配置

部署完成后，需要在云开发控制台配置数据库权限：

1. 进入云开发控制台
2. 点击 "数据库" 选项
3. 创建集合 `bird_observations`
4. 设置集合权限：
   - 读权限：所有人
   - 写权限：仅创建者和管理员

## 错误排查

### 常见错误及解决方案：

1. **错误码 -601034**: "没有权限，请先开通云开发或者云托管"
   - 解决方案：确认已在微信公众平台开通云开发功能

2. **上传失败**: "网络连接超时"
   - 解决方案：检查网络连接，重试上传

3. **依赖安装失败**
   - 解决方案：检查 `package.json` 文件，确保 `wx-server-sdk` 版本正确

4. **函数调用失败**: "找不到函数"
   - 解决方案：确认函数已成功部署，函数名拼写正确

## 使用建议

- 在生产环境部署前，建议先在测试环境中验证所有云函数
- 定期监控云函数的调用次数和执行时间
- 根据实际使用情况调整云函数的内存和超时配置
- 保留本地备份以防云端部署出现问题

## 注意事项

- 部署时会覆盖云端已有的同名函数
- 建议在部署前测试函数功能是否正常
- 云函数有调用频率限制，请合理设计调用逻辑
- 敏感操作（如数据删除）请添加权限验证
