# 地图标记图片显示 — 最终决议

> 状态:已采纳(2026-05)
> 适用范围:`miniprogram/` 下所有鸟类地图标记与详情弹窗的图片展示
> 关联归档:见 `docs/archive/2026-image-display-debug-saga/README.md`

## 1. 最终方案(权威实现)

- **地图标记**:使用微信小程序 `<map>` 组件的**系统默认标记**(蓝色圆点),不试图把本地图片嵌进 marker icon。
- **物种识别**:在每个标记的 `callout.label` / `title` 字段写一个**简短文字标签**(中文名 / 学名 / 关键提示),点击后弹出详情。
- **详情弹窗图片**:物种大图走 `miniprogram/utils/birdData.js` 中 `imageUrl` 字段指向的**项目内本地图片**(例如 `/images/alba.jpg`),直接 `<image src="...">` 渲染。
- **本地资源兜底**:如果 `imageUrl` 缺失或加载失败,使用 `miniprogram/images/default-bird.jpg` 兜底。

## 2. 为什么不再用自定义 marker icon

历史上我们尝试过以下做法,均被放弃:

- **方案 A — 本地 webp/jpg 当 marker icon**:微信小程序的 `<map>` 组件对自定义 marker 图标的本地资源路径支持很挑剔,在某些基础库版本下会显示成空白。反复修了几次仍不可靠。
- **方案 B — 在线图片 URL 当 marker icon**:网络抖动 + 小程序对部分外链域名有限制,标记图标在弱网 / 飞行模式下消失,影响"打开就能看"的体验。
- **方案 C — 用红色背景的"egret"文字标签**:作为过渡方案虽然解决了可见性问题,但只能标识单一种鸟,扩展性差,且偏离了"地图标记 = 通用标记"的范式。

## 3. 当前实现的稳定性来源

- **标记层只用系统样式 + 文字标签** — 不依赖网络,marker 一定可见。
- **图片层只在详情弹窗加载** — 加载失败有兜底,且只在用户主动点击后才发生,不阻塞首屏。
- **数据层统一在 `birdData.js`** — 任何新鸟只需加一条记录,`imageUrl` 指向已存在的本地图片即可。

## 4. 后续约束

- 不要再尝试给 marker 装自定义图片。如果新场景必须区分物种(例如显示不同颜色),用 `callout` 文字 + 颜色 class 实现,不要碰 icon 字段。
- 新增物种时,`imageUrl` 必须使用 `miniprogram/images/` 下已存在的本地文件;不要临时外链,等 PR 合入时把图片一起提交。
- 兜底图 `default-bird.jpg` **不要删**;它是图片缺失 / 加载失败时的最后保险。
